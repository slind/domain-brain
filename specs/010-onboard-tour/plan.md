# Implementation Plan: Domain README — Consolidate Command

**Branch**: `010-onboard-tour` | **Date**: 2026-03-18 | **Spec**: [spec.md](spec.md)

## Summary

Implement a `/consolidate` command that generates and maintains `domain/README.md` — a static, human-readable Markdown document summarising the domain's identity, exposed interfaces, intended usage, and top backlog priorities. The command reads three source files (`config/identity.md`, `distilled/interfaces.md`, `distilled/backlog.md`), composes the README, overwrites any existing version, and appends a changelog entry. No user interaction is required during the run.

## Technical Context

**Language/Version**: Markdown (no programming language) + Claude AI (claude-sonnet-4-6+)
**Primary Dependencies**: Built-in tools only: Read, Write, Glob
**Storage**: Reads `config/identity.md`, `distilled/interfaces.md`, `distilled/backlog.md`; writes `domain/README.md` and appends `distilled/changelog.md`
**Testing**: Manual — invoke `/consolidate` and inspect generated README for section completeness and accuracy
**Target Platform**: Claude AI assistant extension (command file in `.claude/commands/`)
**Project Type**: CLI command file
**Performance Goals**: Single invocation, no follow-up prompts; completes in one AI round-trip
**Constraints**: No external services; no modification of distilled knowledge files; write only to `domain/README.md` and `distilled/changelog.md`
**Scale/Scope**: One domain brain instance per invocation; reads 3 files, writes 2 files
